# SkipCloud Copilot Documentation Hub

This folder now keeps only the markdown files that are useful during day-to-day development and Copilot-assisted coding.

## Keep these docs

| Purpose | Files |
| --- | --- |
| Project orientation | `PROJECT_OVERVIEW.md`, `CODEBASE_MAP.md` |
| Architecture and feature flow | `ARCHITECTURE_ANALYSIS.md`, `FEATURE_FLOW.md` |
| Route and service behavior | `API_DOCUMENTATION.md` |
| Validation and testing guidance | `TESTING_STRATEGY.md` |
| Optional implementation specs | `specs/` |

## Copilot loading

- Project-wide Copilot guidance lives in `.github/copilot-instructions.md`.
- File-specific Copilot guidance lives in `.github/instructions/`.
- The files in this folder are reference documents used by those instruction files.

## Maintenance rule

- Add markdown here only if it helps development, debugging, testing, architecture understanding, or implementation planning.
- Avoid storing presentation, showcase, or retrospective-only documents in this folder.
